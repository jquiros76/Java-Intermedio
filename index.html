Clases & Objetos

¿Qué es una Clase?

Una clase es un molde o plantilla que define atributos (datos) y métodos (comportamiento). Un objeto es una instancia concreta de esa clase.

Clase
El plano / molde (ej: Persona)
Objeto
La instancia real (ej: persona1)
Atributo
Variable de la clase (nombre, edad)
Método
Comportamiento (saludar(), calcular())
Constructor
Método especial para crear objetos
new
Palabra clave para instanciar

Definir una Clase

Persona.java
public class Persona {

    // Atributos
    String nombre;
    int edad;
    String email;

    // Constructor
    public Persona(String nombre, int edad, String email) {
        this.nombre = nombre;
        this.edad = edad;
        this.email = email;
    }

    // Método
    public void presentarse() {
        System.out.println(
            "Hola, soy " + nombre + " y tengo " + edad + " años."
        );
    }
}

// Crear objetos
Persona p1 = new Persona("Ana", 25, "ana@email.com");
Persona p2 = new Persona("Luis", 30, "luis@email.com");

p1.presentarse(); // Hola, soy Ana y tengo 25 años.
p2.presentarse(); // Hola, soy Luis y tengo 30 años.

La palabra clave this

this hace referencia al objeto actual. Se usa cuando el nombre del parámetro es igual al del atributo.

💡 Regla
this.atributo = el atributo de la clase | atributo (sin this) = el parámetro

Encapsulamiento

¿Qué es Encapsulamiento?

Ocultar los datos internos y controlar el acceso mediante getters y setters. Los atributos se declaran private y se acceden mediante métodos públicos.

ModificadorClasePaqueteSubclaseMundo
private✅❌❌❌
default✅✅❌❌
protected✅✅✅❌
public✅✅✅✅

Getters & Setters

CuentaBancaria.java
public class CuentaBancaria {

    private String titular;
    private double saldo;

    public CuentaBancaria(String titular, double saldoInicial) {
        this.titular = titular;
        this.saldo = saldoInicial;
    }

    // Getter
    public double getSaldo() {
        return saldo;
    }

    // Setter con validación
    public void depositar(double monto) {
        if (monto > 0) {
            saldo += monto;
            System.out.println("Depósito exitoso. Saldo: " + saldo);
        } else {
            System.out.println("Monto inválido");
        }
    }

    public void retirar(double monto) {
        if (monto > 0 && monto <= saldo) {
            saldo -= monto;
        } else {
            System.out.println("Fondos insuficientes");
        }
    }
}

// Uso
CuentaBancaria cuenta = new CuentaBancaria("Carlos", 1000.0);
cuenta.depositar(500);
System.out.println(cuenta.getSaldo()); // 1500.0
// cuenta.saldo = 999999; ← ERROR: saldo es private
💡 Beneficio
El encapsulamiento protege los datos de cambios no autorizados y permite agregar validaciones en los setters.

TEMA 03 / 07
Herencia

¿Qué es la Herencia?

Una clase hija puede heredar atributos y métodos de una clase padre, reutilizando código. Se usa la palabra clave extends.

Animal (Padre)
↓ extends
Perro
Gato
Pájaro

Ejemplo de Herencia

Herencia.java
// Clase Padre
public class Animal {
    protected String nombre;
    protected int edad;

    public Animal(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    public void comer() {
        System.out.println(nombre + " está comiendo");
    }
}

// Clase Hija
public class Perro extends Animal {

    private String raza;

    public Perro(String nombre, int edad, String raza) {
        super(nombre, edad); // llama al constructor del padre
        this.raza = raza;
    }

    public void ladrar() {
        System.out.println(nombre + " dice: ¡Guau!");
    }
}

Perro perro = new Perro("Rex", 3, "Labrador");
perro.comer();  // Heredado de Animal → Rex está comiendo
perro.ladrar(); // Propio de Perro → Rex dice: ¡Guau!
📌 super
super() llama al constructor del padre. super.metodo() llama a un método del padre.

TEMA 04 / 07
Polimorfismo

¿Qué es el Polimorfismo?

Un objeto puede tomar múltiples formas. Se logra mediante @Override (sobreescritura) en las clases hijas.

Override — Sobreescribir métodos

Polimorfismo.java
class Forma {
    public double area() { return 0; }
    public void describir() {
        System.out.println("Área: " + area());
    }
}

class Circulo extends Forma {
    private double radio;
    public Circulo(double r) { this.radio = r; }

    @Override
    public double area() { return Math.PI * radio * radio; }
}

class Rectangulo extends Forma {
    private double ancho, alto;
    public Rectangulo(double a, double b) { ancho=a; alto=b; }

    @Override
    public double area() { return ancho * alto; }
}

// Polimorfismo en acción
Forma[] formas = {
    new Circulo(5),
    new Rectangulo(4, 6)
};

for (Forma f : formas) {
    f.describir(); // Cada uno calcula SU área
}
🎯 Clave
La referencia es del tipo padre (Forma), pero el comportamiento en tiempo de ejecución es el del hijo. Esto se llama dynamic dispatch.

TEMA 05 / 07
Interfaces & Clases Abstractas

Interface

Un contrato que define QUÉ debe hacer una clase, sin implementar el cómo. Una clase puede implementar múltiples interfaces.

Interfaces.java
// Interface = contrato
public interface Pagable {
    void procesarPago(double monto);
    boolean verificarFondos();
}

public interface Notificable {
    void enviarNotificacion(String mensaje);
}

// Implementación múltiple
public class TarjetaCredito implements Pagable, Notificable {
    private double limite;

    @Override
    public void procesarPago(double monto) {
        System.out.println("Pagando " + monto + " con tarjeta");
    }

    @Override
    public boolean verificarFondos() { return limite > 0; }

    @Override
    public void enviarNotificacion(String msg) {
        System.out.println("SMS: " + msg);
    }
}

Diferencia: Interface vs Clase Abstracta

InterfaceClase Abstracta
Herenciaimplements (múltiple)extends (una sola)
MétodosSolo abstractos (default en Java 8+)Abstractos y concretos
AtributosSolo constantesVariables de instancia
ConstructorNo tieneSí tiene
Uso idealDefinir un contratoCompartir código base

TEMA 06 / 07
Colecciones (ArrayList & HashMap)

ArrayList — Lista dinámica

Como un array pero con tamaño dinámico. Permite agregar, eliminar y acceder por índice.

Colecciones.java
import java.util.ArrayList;

ArrayList<String> tareas = new ArrayList<>();

// Agregar
tareas.add("Diseñar base de datos");
tareas.add("Crear API REST");
tareas.add("Hacer pruebas");

// Acceder
System.out.println(tareas.get(0));    // Diseñar base de datos
System.out.println(tareas.size());   // 3

// Eliminar
tareas.remove("Hacer pruebas");

// Recorrer
for (String t : tareas) {
    System.out.println("- " + t);
}

HashMap — Clave-Valor

HashMap.java
import java.util.HashMap;

HashMap<String, Integer> edades = new HashMap<>();

// Agregar pares clave-valor
edades.put("Ana", 25);
edades.put("Luis", 30);
edades.put("María", 28);

// Acceder por clave
System.out.println(edades.get("Ana"));       // 25
System.out.println(edades.containsKey("Luis")); // true

// Recorrer
for (String nombre : edades.keySet()) {
    System.out.println(nombre + " → " + edades.get(nombre));
}

Manejo de Excepciones

try-catch-finally

Las excepciones son errores en tiempo de ejecución. Java permite capturarlos para que el programa no se detenga abruptamente.

Excepciones.java
public class Excepciones {
    public static void main(String[] args) {

        try {
            int resultado = 10 / 0;  // ArithmeticException
            System.out.println(resultado);
        } catch (ArithmeticException e) {
            System.out.println("Error: división por cero");
            System.out.println(e.getMessage());
        } finally {
            System.out.println("Bloque finally: siempre se ejecuta");
        }

        // Múltiples catch
        try {
            String s = null;
            s.length(); // NullPointerException
        } catch (NullPointerException e) {
            System.out.println("Objeto nulo");
        } catch (Exception e) {
            System.out.println("Error general: " + e.getMessage());
        }
    }
}

Excepciones más comunes

ExcepciónCausa
NullPointerExceptionUsar un objeto null
ArrayIndexOutOfBoundsExceptionÍndice fuera de rango
ArithmeticExceptionDivisión por cero
NumberFormatExceptionConvertir "abc" a número
ClassCastExceptionCast inválido entre tipos

Crear excepciones personalizadas

CustomException.java
public class SaldoInsuficienteException extends Exception {
    public SaldoInsuficienteException(String mensaje) {
        super(mensaje);
    }
}

// Uso con throws
public void retirar(double monto) throws SaldoInsuficienteException {
    if (monto > saldo) {
        throw new SaldoInsuficienteException("Saldo insuficiente: " + saldo);
    }
    saldo -= monto;
}
